01Matrices as machines

A matrix is a machine that takes a vector in, processes it, and hands a vector back — with the one restriction that the processing is linear.

Start from an ordinary function. f(x) = 2x takes a number, doubles it, returns a number. Now let the input be a list of numbers (a vector) and the output also a list. A matrix is exactly that machine, restricted to be linear, meaning it obeys two rules — "the machine plays no tricks":

Additivity & homogeneity (the definition of linear) f(u + v) = f(u) + f(v)   and   f(c·v) = c·f(v)

We obsess over linear machines because they are the simplest non-trivial functions, they are everywhere (rotations, scalings, projections, the Wx inside every neural-net layer), and — the punchline — any linear machine on finite vectors is fully described by a finite grid of numbers. That grid is the matrix. We don't invent matrices; they fall out as the minimum data needed to pin down a linear map.

Shape = the dimensions it maps between

An m × n matrix accepts an n-dimensional input and returns an m-dimensional output. It maps ℝⁿ → ℝᵐ.

Read the shape this way, always(rows, columns) = (out, in)  ·  columns eat the input, rows build the output

That is the "changes dimensions" idea: a 3072 × 768 weight matrix takes a 768-dim token and blows it up to 3072 dims.

Why a matrix is "a collection of vectors"

This is the master key. A linear machine is completely determined by where it sends the basis vectors, because every vector is a combination of basis vectors and linearity lets you push the machine through the combination:

v = x·e₁ + y·e₂   ⟹   T(v) = x·T(e₁) + y·T(e₂)

So if you know where e₁ and e₂ land, you know where every vector lands. Store T(e₁) as column 1, T(e₂) as column 2. That is the whole matrix.

The one sentence to memorizeThe columns of a matrix are the landing spots of the input space's basis vectors. Everything else (eigenvectors, rank, SVD) builds on this.

02Matrix multiplication — both methods

Matrix × vector

Because the columns are landing spots, the multiplication rule is forced, not arbitrary.

Method 1 — column picture (linear combination of columns)

The output is the input's entries used as weights on the columns of A.

A·v = v₁·(col 1) + v₂·(col 2) + …

Method 2 — row picture (dot product per output coordinate)

Each output entry is the dot product of a row of A with v. This is the "system of equations" view: each row is one equation, one measurement.

Worked example
A = [2 0] v = [4] [1 3] [5]
Columns: A·v = 4·[2,1] + 5·[0,3] = [8,4] + [0,15] = [8, 19]
Rows: out₁ = [2,0]·[4,5] = 8 ; out₂ = [1,3]·[4,5] = 4+15 = 19 ⟹ [8, 19] ✓ same

Column view tells you what outputs are reachable (the span of the columns). Row view is the lens for solving systems. Both must agree — same operation, two angles.

Matrix × matrix

Two distinct reasons you need it:

  • Composition — "do B, then A" is itself one machine, and its matrix is AB:  (AB)v = A(Bv).
  • Batching — stack many input vectors as columns of B; then AB applies the same transform to all of them at once (this is XW over a batch).
Shape rule — inner dimensions must match (m × k)(k × n) = (m × n)  ·  order matters: AB ≠ BA in general
Worked example — both methods
A = [1 2] B = [5 6] [3 4] [7 8]
Method 1 — dot product  (entry i,j = row i of A · col j of B)
(1,1)=[1,2]·[5,7]=19 (1,2)=[1,2]·[6,8]=22 (2,1)=[3,4]·[5,7]=43 (2,2)=[3,4]·[6,8]=50 AB = [19 22] [43 50]
Method 2 — column method  (each output column = A acting on that column of B)
col1 = A·[5,7] = 5·[1,3]+7·[2,4] = [19,43] col2 = A·[6,8] = 6·[1,3]+8·[2,4] = [22,50] ✓ same matrix
Bonus view that runs your stackA third reading is AB as a sum of outer products (column of A times row of B, summed). This is the entire idea behind low-rank decomposition and LoRA: replace a big update with B·A where the inner dimension r is tiny, i.e. a short sum of rank-1 layers.

03Eigenvalues & eigenvectors

Most vectors get rotated and stretched by a matrix. Eigenvectors are the special directions that only get stretched — no turning.

Picture stretching a rubber sheet diagonally. Most painted arrows swing to point somewhere new. The arrows lying along the pure stretch and squeeze axes just get longer or shorter without turning. Those are the eigenvectors; the stretch factors are the eigenvalues.

The defining equation — A only scales v A·v = λ·v   (v ≠ 0)

An eigenvector is really a direction: if v works, so does any multiple of it, with the same λ. That is why we find a line (and often normalize it).

Why we care

Eigenvectors are the matrix's natural axes — the coordinate system in which the machine stops mixing and is pure stretching. Concretely:

  • Repeated application is trivial: Aᵏv = λᵏv. Governs Markov chains, PageRank, dynamics — the largest |λ| dominates (>1 blow-up, <1 decay, =1 steady state).
  • Stability & conditioning: the Hessian's eigenvalues are curvatures; the ratio largest/smallest is the condition number that makes gradient descent zig-zag.
  • PCA is the eigendecomposition of the covariance matrix — eigenvectors are the directions of maximum variance, eigenvalues are how much.

How to find them, step by step

A·v = λ·v ⟹ (A − λI)·v = 0

We need a nonzero v crushed to zero by (A − λI). A matrix that crushes a nonzero vector is singular, which happens exactly when its determinant is zero (the determinant is the volume-scaling factor; zero means a dimension collapsed):

The characteristic equation det(A − λI) = 0
  1. Form A − λI.
  2. Set its determinant to zero → polynomial in λ.
  3. Solve for λ (the eigenvalues).
  4. For each λ, solve (A − λI)v = 0 for the eigenvector(s).
Worked example
A = [2 1] [1 2]
det: (2−λ)² − 1 = 0 ⟹ (2−λ)² = 1 ⟹ 2−λ = ±1 ⟹ λ = 1 or 3
λ = 3: (A−3I)=[[−1,1],[1,−1]] ⟹ −v₁+v₂=0 ⟹ eigenvector [1, 1]
λ = 1: (A−1I)=[[1,1],[1,1]] ⟹ v₁+v₂=0 ⟹ eigenvector [1, −1]
Check: A·[1,1]=[3,3]=3·[1,1] ✓   A·[1,−1]=[1,−1]=1·[1,−1] ✓
The two eigenvectors are perpendicular — not luck. Symmetric matrices always have orthogonal eigenvectors and real eigenvalues.
TrapA pure rotation turns every vector, so it has no real eigenvectors; its eigenvalues come out complex. The math is correctly saying "this machine only spins, it has no stretch axes."

04Pivots

A completely separate idea from eigenvalues. When you row-reduce a matrix into staircase (echelon) form, the pivot is the leading nonzero entry of each row — the anchor you eliminate with. Columns with a pivot are pivot columns; columns without one are free columns.

Worked elimination
A = [1 2 1] R2 ← R2 − 2R1 : [0 0 3] [2 4 5] R3 ← R3 − R1 : [0 0 2] [1 2 3] R3 ← R3 − ⅔R2 : [0 0 0] final: [1 2 1] ← pivot in column 1 [0 0 3] ← pivot in column 3 [0 0 0]
Two pivots (columns 1, 3). Column 2 is free (it was just twice column 1). Rank = 2, nullity = 1.

Where pivots are used

  • Number of pivots = rank — the true working dimension.
  • Pivot columns form a basis for the column space; free columns count the null space.
  • Solvability: unique / infinite / no solution depending on pivots and free columns.
  • Invertibility: an n×n matrix is invertible iff it has n pivots.
  • Determinant = product of pivots (with sign for row swaps); pivots are the diagonal of U in LU.
Trap — pivots ≠ eigenvaluesBoth relate to the determinant, so people equate them. They are not equal term by term. They only agree on the product:  ∏ pivots = ±det = ∏ eigenvalues.

05Rank & low rank

Rank is the number of genuinely independent directions a matrix carries — the dimension of its column space.

A 3×3 matrix whose three columns all lie in a plane has rank 2: it's a 2D machine wearing a 3D costume. Rank is the matrix's true working dimension. All of these are the same number:

independent columnsindependent rowspivotsnonzero singular valuesdim of column space

Full rank = the maximum min(m, n). For a square matrix that means invertible, nonzero determinant, no zero eigenvalues. Low rank (rank deficient) = some columns are redundant; the matrix collapses dimensions and loses information.

How a matrix becomes low rank — the outer product

Rank-1 construction
u = [1,2,3]ᵀ , v = [1,0,2]ᵀ u·vᵀ = [1 0 2] [2 0 4] every column is a multiple of u [3 0 6] ⟹ ONE independent direction ⟹ rank 1

The key fact: a sum of r rank-1 outer products has rank at most r. So "low rank" literally means "buildable from a few underlying factors."

Where rank shows up in ML LoRA: learn ΔW = B·A with tiny inner rank r instead of a full update — fine-tuning is a small low-rank nudge.
Recommendation: the user×item matrix R ≈ U·Vᵀ is low rank because ratings come from a few latent factors.
Compression / PCA: keep the top-r components for the best rank-r approximation (Eckart-Young) — signal lives in the large components, noise in the small.

06Special matrices

Identity I

The do-nothing machine: Iv = v. Multiplicative identity, full rank, every eigenvalue 1, every vector an eigenvector. Shows up as residual connections (x + f(x)) and the λI in ridge regression.

Singular matrix

Square but not invertible: det = 0, rank < n, has 0 as an eigenvalue, nonempty null space. Information is lost. The failure mode behind collinear features wrecking regression.

Symmetric matrix (A = Aᵀ) — the important one

The matrix of mutual relationships and second-order structure: covariances, Gram matrices XᵀX, kernels, Hessians, undirected-graph adjacencies. Mutuality forces symmetry; symmetry pays back the cleanest eigenstructure:

  • All eigenvalues are real.
  • Eigenvectors are orthogonal (a full orthonormal set exists).
  • Spectral theorem: A = QΛQᵀ with Q orthogonal, Λ diagonal.
Read it as a sentenceA = Q Λ Qᵀ   =   rotate into the eigenbasis · scale · rotate back  (rigidly, no distortion)

Orthogonal matrix (QᵀQ = I)

Inverse equals transpose; columns orthonormal; preserves all lengths and angles (a rigid motion). Every eigenvalue has magnitude 1. Parent of rotations and reflections.

Projection matrix (P² = P)

Squashes vectors onto a subspace. Projecting twice = once (idempotent). Eigenvalues only 0 (killed) and 1 (kept). Mental image: a shadow on the ground.

Orthogonal projection onto the column space of A — this IS linear regressionP = A (AᵀA)⁻¹ Aᵀ  (the "hat matrix" that produces ŷ)

Reflection matrix (R² = I)

A mirror. Reflect twice → back to start, so R = R⁻¹. Eigenvalues +1 (on the mirror) and −1 (perpendicular, flipped). Determinant −1 (orientation reversed). Householder reflections power QR decomposition.

Markov (stochastic) matrix

Entries ≥ 0, each column sums to 1 (probability conserved). Largest eigenvalue is exactly 1, and its eigenvector is the stationary distribution; all other |λ| ≤ 1, so iteration converges. This is Markov chains, HMMs, and PageRank.

Steady state
P = [0.9 0.5] solve P·s = s [0.1 0.5] ⟹ s = [5/6, 1/6]

07Positive definite & positive semidefinite

These are symmetric matrices with a sign condition. They are the algebraic language of "a bowl," which is why they run convex optimization.

For a symmetric A PD:   vᵀA v > 0 for all v ≠ 0   |   PSD:   vᵀA v ≥ 0 for all v

The quantity vᵀAv is a bowl-shaped function of v. PD = the bowl curves strictly up in every direction (single lowest point). PSD = up or flat. Indefinite = a saddle. Rotate into the eigenbasis and the reason is obvious:

vᵀA v = λ₁y₁² + λ₂y₂² + … + λₙyₙ²
  • all λ > 0 → always positive → PD
  • some λ = 0 → flat along that axis → PSD
  • some λ < 0 → curves down → indefinite / saddle
Why this is everywhere in ML The Hessian is curvature: PD ⟹ strict local minimum, PSD everywhere ⟹ the function is convex (single global min), indefinite ⟹ a saddle point.
Covariance and Gram matrices are always PSD by a one-liner:  vᵀ(XᵀX)v = (Xv)ᵀ(Xv) = ‖Xv‖² ≥ 0. It is PD when X has independent columns (full column rank). Collinear features ⟹ only PSD & singular ⟹ add λI (ridge), which bumps every eigenvalue up and restores PD + invertibility.

How to test PD / PSD

all eigenvalues > 0 / ≥ 0all pivots positiveleading minors > 0 (Sylvester)writable as BᵀB

08Why decompose a matrix at all

A matrix tangles rotation, stretch, shear and collapse together. A decomposition factors that one machine into a product of simpler machines, each doing one clean thing. The analogy is number factoring: 60 = 2²·3·5 instantly reveals structure. Factoring a matrix gives easy solving (triangular/diagonal), easy powers, transparency of what it does, compression, and numerical stability.

09LU decomposition

A = LU: L lower-triangular (1s on the diagonal), U upper-triangular. It is just Gaussian elimination recorded as a product — U is the staircase, L stores the multipliers used to clear each column. The diagonal of U is the pivots; their product is the determinant.

Payoff: solving Ax = b becomes two trivial triangular solves — set Ux = y, solve Ly = b top-down (forward substitution), then Ux = y bottom-up (back substitution). Each is O(n²) and reusable for any new b. With row swaps it becomes PA = LU.

10Diagonalization = eigenvalue decomposition

P = eigenvectors as columns, D = eigenvalues on the diagonalA = P D P⁻¹

Why this form is forced (derivation)

AP = A[v₁|…|vₙ] = [Av₁|…|Avₙ] = [λ₁v₁|…|λₙvₙ] = P D  ⟹  A = P D P⁻¹

Read it right-to-left as a three-step machine

  1. P⁻¹ — change coordinates into the eigenbasis (express input as amounts of each eigenvector).
  2. D — scale each eigen-coordinate by its eigenvalue (pure, independent stretching).
  3. P — change back to the standard basis.

The complexity of A was an illusion of the coordinate system; in the eigenbasis it is just scaling.

The payoff — powers collapseAᵏ = P Dᵏ P⁻¹  (all the inner P⁻¹P cancel; Dᵏ just powers each diagonal entry)
Worked example
A = [2 1] P = [1 1] D = [3 0] [1 2] [1 −1] [0 1] AP = [3 1] PD = [3 1] ✓ AP = PD [3 −1] [3 −1]
Limitations (the motivation for SVD)Not every matrix is diagonalizable — you need n independent eigenvectors (e.g. [[1,1],[0,1]] fails), and rectangular matrices have none. For non-symmetric A, P is not orthogonal, so the change of basis is skewed and numerically fragile. The clean case: symmetric A gives A = QΛQᵀ with rigid orthogonal Q.

11Singular value decomposition

SVD does for every matrix what the spectral theorem does for symmetric ones — rectangular, singular, non-diagonalizable, anything.

U, V orthogonal · Σ holds the singular values σ₁ ≥ σ₂ ≥ … ≥ 0A = U Σ Vᵀ

The geometry — circle becomes ellipse

Every linear map, however ugly, does exactly three things (right to left): Vᵀ a rigid rotation of the input, Σ an axis-aligned stretch, U a rigid rotation of the output. Rotate, stretch, rotate.

How it is built on eigendecomposition

AᵀA = (UΣVᵀ)ᵀ(UΣVᵀ) = V Σᵀ(UᵀU)Σ Vᵀ = V (Σ²) Vᵀ

That is exactly the eigendecomposition of the symmetric PSD matrix AᵀA: the right singular vectors V are its eigenvectors, and σᵢ = √λᵢ (real and ≥ 0 because AᵀA is PSD). Similarly AAᵀ = U(Σ²)Uᵀ gives U.

  1. Form AᵀA; eigendecompose → V and λᵢ.
  2. σᵢ = √λᵢ, sorted descending → Σ.
  3. uᵢ = A vᵢ / σᵢ → U.
Worked example — and it shows low rank
A = [1 1] AᵀA = [2 2] eigenvalues 4, 0 [1 1] [2 2] ⟹ σ₁ = 2, σ₂ = 0 ⟹ rank 1 v₁ = [1,1]/√2 , u₁ = A v₁/σ₁ = [1,1]/√2 A = σ₁·u₁v₁ᵀ = 2·½[[1,1],[1,1]] = [[1,1],[1,1]] ✓ rebuilt from ONE rank-1 layer
The general form — a sum of rank-1 layers, biggest first A = σ₁u₁v₁ᵀ + σ₂u₂v₂ᵀ + … + σᵣuᵣvᵣᵀ

Truncating after r terms gives the provably best rank-r approximation (Eckart-Young). Big σ = signal, small σ = detail/noise. This is the engine of PCA, image compression, LSA, recommendation, and denoising. (LoRA assumes the update is one of these short sums from the start.)

SVD vs eigendecomposition

 Eigendecomposition A = PDP⁻¹SVD A = UΣVᵀ
Exists forsquare & diagonalizable onlyevery matrix, any shape
BasesP generally skewedU, V both orthonormal (rigid)
Framesone shared basis (P, P⁻¹)two bases — V in, U out

They coincide for a symmetric PSD matrix (U = V = Q, Σ = Λ). SVD is the spectral theorem extended to all matrices.

Why SVD is the workhorseAlways exists · best low-rank approximation (PCA = SVD of centered data) · numerically stable (how libraries compute rank, condition number, least squares) · gives the pseudo-inverse · reveals rank and the four subspaces · condition number = σ_max / σ_min.

12Basis

A basis is the smallest set of building-block directions from which every vector is built in exactly one way. Two conditions produce that uniqueness:

  • Spanning — they reach everything.
  • Independence — no redundancy.

Too few ⟹ unreachable vectors; too many (dependent) ⟹ non-unique recipes. The count of basis vectors is the dimension, and it is fixed for a space. Any n independent vectors in ℝⁿ form a basis. A basis is a language for describing vectors, and coordinates are basis-dependent — changing basis is translation (exactly what P⁻¹ and Vᵀ did). The pivot columns of a matrix form a basis for its column space, so basis, span, independence, rank, and dimension are five views of one structure. PCA is literally the act of choosing a better basis — the one aligned with variance.

13Inverse & invertibility

The inverse A⁻¹ is the machine that undoes A: A⁻¹A = I. You can run a machine backwards only if it is a perfect one-to-one correspondence — every output came from exactly one input.

The logical heart of non-invertibilityIf A is not invertible there is a nonzero v with Av = 0. Then for any x:  A(x + v) = Ax + Av = Ax. Two different inputs give the same output, so the machine cannot be reversed. A nonzero null space = inputs colliding = no inverse.

For a square n×n matrix these are all the same fact ("nothing collapsed"):

full rank ncolumns independent (a basis)columns span ℝⁿnull space = {0}unique solution for every bdet ≠ 0no zero eigenvaluesno zero singular valuesn pivots

Why the determinant test works: det is the signed volume-scaling factor. det ≠ 0 ⟹ volume survived, no dimension flattened, reversible. det = 0 ⟹ volume collapsed, a dimension was crushed, not reversible.

2×2 inverse
A = [a b] A⁻¹ = (1/det)[ d −b] det = ad − bc [c d] [−c a] A = [2 1] , det = 3 : A⁻¹ = ⅓[ 2 −1] [1 2] [−1 2] check: A·A⁻¹ = ⅓[[3,0],[0,3]] = I ✓

A rectangular matrix has no true two-sided inverse (it changes the dimension of the space) — which is why we need the pseudo-inverse.

14Pseudo-inverse

The best-effort inverse for matrices that aren't invertible (rectangular or singular). Two cases:

  • Overdetermined (tall, more equations than unknowns): usually no exact solution — the pseudo-inverse returns the least-squares solution, the x minimizing ‖Ax − b‖². This is linear regression; AA⁺ projects b onto the column space.
  • Underdetermined (wide, infinitely many solutions): it picks the minimum-norm solution.
Computed cleanly through SVDA⁺ = V Σ⁺ Uᵀ   — invert each nonzero σ (σ → 1/σ), leave zeros as zeros

Intuition straight from the rotate-stretch-rotate picture: to undo A, reverse-rotate (Uᵀ), un-stretch the directions that were stretched (Σ⁺), reverse-rotate (V). A direction with σ = 0 was crushed flat and cannot be un-crushed (no 1/0), so you zero it — that dropped term is the source of "best effort."

Worked example
A = [1 1] (σ₁=2, σ₂=0, u₁=v₁=[1,1]/√2) [1 1] A⁺ = (1/σ₁)·v₁u₁ᵀ = ¼[[1,1],[1,1]] AA⁺ = [[.5,.5],[.5,.5]] = projection onto the line [1,1] (the column space) ✓

If A is invertible, A⁺ = A⁻¹ — the pseudo-inverse strictly generalizes the inverse. The regression normal equations XᵀX β = Xᵀy are just (XᵀX)⁻¹Xᵀ written out.

15Null space

The null space of A is the set of all vectors v with Av = 0 — the inputs the matrix crushes to zero.

How to get it: solve Av = 0 by elimination; non-pivot columns become free variables; solve the pivot variables in terms of them; each free variable yields one basis vector of the null space.

Rank–nullity theoremrank + nullity = number of columns

Why it matters: a nonzero null space means lost information and no inverse (two inputs collide); it is the set of directions your features cannot distinguish (collinearity), the directions a pseudo-inverse discards, and the source of non-unique solutions that regularization resolves.

17The single thread

Matrices are machines whose columns are basis landing spots. Matrix multiplication chains machines or runs one over a batch. Eigenvectors are a matrix's natural stretch axes; rank counts its real independent directions. Symmetric matrices factor into a rigid rotate-scale-rotate, and "symmetric with non-negative eigenvalues" (PSD) is the algebraic signature of a bowl — the language of convex optimization. SVD unifies all of it: every matrix is a rotation, a stretch, and a rotation, with the stretches ordered so you can keep the big ones and throw the rest away. Invertibility is the single statement that a machine collapses nothing; the pseudo-inverse is what you reach for when it does.
